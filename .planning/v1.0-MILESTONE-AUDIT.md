## Unsatisfied Requirements

### DATA-07: Pipeline rejects duplicate enqueue for .transcribing meetings
- **Phase:** 05 — Pipeline Data Integrity
- **Issue:** `TranscriptionPipeline.enqueue()` line 85 checks `status == .done` only. A meeting actively processing (dequeued from queue, status=.transcribing in DB) would pass the check and be double-queued.
- **Fix:** Add `|| status == .transcribing` to DB status check. Add test `testRejectsDuplicateEnqueueForTranscribingMeeting`.
- **Severity:** Low — requires exact race timing (meeting detected twice while already transcribing)

### ERR-03 (partial): MicrophoneCapture weak self
- **Phase:** 06 — Error Discipline
- **Issue:** `MicrophoneCapture.swift` line 71 uses `self?.processInputBuffer()` — silent optional chaining without guard let self. Was not in plan 06-02's files_modified list.
- **Fix:** Replace with `guard let self else { return }` (real-time thread, no logging — same pattern as AudioRecorder).
- **Severity:** Low — real-time callback, logging would cause priority inversion anyway

### DATA-01 (partial): DB insert failure user notification

- **Issue:** Abort path wired via state machine (coordinator catch → .error). Notification component was deferred to Phase 8 (UX-03) which is now complete. Needs human verification of full path.
- **Severity:** Info — likely works, just needs manual test
