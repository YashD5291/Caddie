Edit project.yml to make three changes:

1. **Swift version bump (BUILD-01):** Change `SWIFT_VERSION: "5.9"` to `SWIFT_VERSION: "6.0"` in `settings.base`. This enables GRDB 7.10 compatibility and strict concurrency checking. Do NOT use 6.1 or 6.2 -- use 6.0 as the minimum that satisfies the requirement while keeping maximum compatibility.

2. **Strict concurrency (BUILD-01):** Add `SWIFT_STRICT_CONCURRENCY: complete` to `settings.base` (immediately after the SWIFT_VERSION line). This enables compile-time data race detection. If `complete` produces too many warnings to build, the executor should downgrade to `targeted` as a stepping stone.

3. **Scheme with selective coverage (BUILD-02):** Add a `schemes` section to project.yml that restricts code coverage instrumentation to the Caddie app target only, excluding SPM C dependencies (yyjson). This fixes the `___llvm_profile_runtime` undefined symbol linker error (Swift bug SR-14788). Add the following top-level section:

```yaml
schemes:
  Caddie:
    build:
      targets:
        Caddie: all
    run:
      config: Debug
    test:
      gatherCoverageData: true
      coverageTargets:
        - Caddie
      targets:
        - CaddieTests
```

The final project.yml should have these sections in order: name, options, settings (with SWIFT_VERSION: "6.0" and SWIFT_STRICT_CONCURRENCY: complete), packages, targets, schemes.

After editing project.yml, regenerate the Xcode project:
```bash
cd /Users/yashdesai/Codebase/Fun/Caddie && xcodegen generate
```
