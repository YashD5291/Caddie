  <action>
Create `scripts/download-models.sh` -- a bash script that downloads all FluidAudio ML model files from HuggingFace into `Resources/Models/`. Per D-01 (build-phase download) and D-02 (not stored in git).

The script must:
1. Use `set -euo pipefail` for strict error handling
2. Use `${SRCROOT}` for the base path (set by Xcode during build) with a fallback: `SRCROOT="${SRCROOT:-$(cd "$(dirname "$0")/.." && pwd)}"` so it also works when run standalone
3. Define `MODELS_DIR="${SRCROOT}/Resources/Models"`
4. Define a `download_file()` function that:
   - Takes 4 args: repo, path, dest, expected_size (optional, default 0)
   - Skips download if file exists AND (expected_size is 0 OR actual size matches expected)
   - Re-downloads if size mismatch (prints "Size mismatch" message)
   - Uses `curl -sL` with the URL pattern `https://huggingface.co/${repo}/resolve/main/${path}`
   - Creates parent directories with `mkdir -p`
   - Uses `stat -f%z` for macOS file size (not `stat -c%s` which is Linux)

5. Download ASR models from repo `FluidInference/parakeet-tdt-0.6b-v3-coreml`:
   - `Encoder.mlmodelc/weights/weight.bin` (expected: 445187200)
   - `Decoder.mlmodelc/weights/weight.bin` (expected: 23604992)
   - `JointDecision.mlmodelc/weights/weight.bin` (expected: 12642764)
   - `Preprocessor.mlmodelc/weights/weight.bin` (expected: 491072)
   - For each of Encoder, Decoder, JointDecision, Preprocessor: download `coremldata.bin`, `metadata.json`, `model.mil` (no size check needed for small files)
   - Create `analytics/` directory for each model component with `mkdir -p`
   - `parakeet_vocab.json` (expected: 151122)

  <action>
Update `project.yml` to add a `preBuildScripts` entry under the `Caddie` target. Per D-01, this script runs the download during xcodebuild.

Add this block under `targets.Caddie`, BEFORE the `sources` key:

```yaml
    preBuildScripts:
      - name: Download ML Models
        script: "${SRCROOT}/scripts/download-models.sh"
        basedOnDependencyAnalysis: false
```

`basedOnDependencyAnalysis: false` ensures the script runs on every build (not just when inputs change), because the script itself is idempotent and fast when models already exist.

Also update the `resources` section to exclude `.mlpackage` files (per Pitfall 2 from RESEARCH.md -- Xcode would try to compile them):

```yaml
    resources:
      - path: Resources
        excludes:
          - "Assets.xcassets"
          - "Models/**/*.mlpackage"
```

After editing, run `xcodegen generate` to verify the project file generates without errors.
