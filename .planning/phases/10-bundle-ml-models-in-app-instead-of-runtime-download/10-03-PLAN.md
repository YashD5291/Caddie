<objective>
Update CI/CD workflows to cache ML models and handle increased build size.

Purpose: The preBuildScript in project.yml will download ~711MB of models during CI builds. Without caching, every CI run would re-download models from HuggingFace, adding 5-10 minutes. This plan adds GitHub Actions cache for the models directory and increases timeouts. Per D-11, D-12.
Output: Updated release.yml and ci.yml with model caching.
</objective>

Make these changes to `.github/workflows/release.yml`:

1. **Increase timeout** from 45 to 60 minutes (D-12):
   Change `timeout-minutes: 45` to `timeout-minutes: 60`

2. **Add model cache step** after "Generate Xcode Project" and BEFORE "Build Release". Insert this step:
```yaml
      - name: Cache ML Models
        uses: actions/cache@v4
        with:
          path: Resources/Models
          key: ml-models-parakeet-v3-sortformer-v2
          restore-keys: |
            ml-models-
```

The cache key `ml-models-parakeet-v3-sortformer-v2` is stable -- it only changes when model versions change. The `restore-keys: ml-models-` prefix allows partial cache hits when one model set is updated but not the other.

On cache hit, the preBuildScript (download-models.sh) will find all files already present and skip all downloads (idempotent). On cache miss, it downloads ~711MB from HuggingFace and the cache saves it for next time.

No other changes to release.yml. The Build Release step's xcodebuild invocation will automatically trigger the preBuildScript which runs download-models.sh.
