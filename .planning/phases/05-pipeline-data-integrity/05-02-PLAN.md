<objective>
Add pipeline entry guards (duplicate rejection, bounded queue) and make retry transcription safe.

Purpose: Currently enqueue() accepts any meeting ID without checking if it is already being processed or done. The queue is unbounded. Retry passes database but does not refresh the connection. These are three independent but related safety valves at the pipeline boundary.

Output:
1. enqueue() checks meeting status in DB and rejects .transcribing/.done (DATA-07)
2. enqueue() rejects when queue depth >= 50 (DATA-08)
3. Retry path verifies WAV exists (already done) and uses coordinator's database (non-optional, always fresh per Phase 4) (DATA-06)
</objective>
