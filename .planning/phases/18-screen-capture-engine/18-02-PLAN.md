<objective>
Implement the live capture path of `ScreenRecorder`: a video-only `SCStream` feeding an `AVAssetWriter` on the dedicated confined writer queue, producing a crash-safe, bitrate-capped HEVC `.mov` with a host-clock anchor. This wires the pure functions and types from Plan 01 into the real hardware path and handles the four sharp edges (first-frame, static-screen, stream death, non-blocking finalize).

Purpose: This is the risk-bearing core of the milestone — the engine that Phase 19 injects into `RecordingCoordinator`. It must never share state with the audio ring buffer and must degrade to a finalize-and-surface on any stream error (Phase 19 owns the audio-only fallback).

Output: A complete `ScreenRecorder.start(target:preset:outputURL:)` / `stop()` implementation with `WriterSink` handling `SCStreamOutput`/`SCStreamDelegate` on `writerQueue`, plus the capture-target API (display OR window) that Phase 21 Settings will drive.

NEVER leave an unfinalized writer on the error path (no silent failure — core value). In Phase 18 the engine's job ends at "finalize + surface"; Phase 19 owns degrade-to-audio-only.

    4. Idempotency: calling stop() after didStopWithError (or twice) must be a safe no-op via the state guard.
