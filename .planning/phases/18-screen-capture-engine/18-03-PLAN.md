<objective>
Build the empirical VID-07 crash-safety gate: a DEBUG-only headless harness that drives `ScreenRecorder` in a separate OS process, plus a shell script that records, `kill -9`s the process mid-recording, and validates that the fragmented `.mov` is still playable and missing at most the last ~10 seconds. This is the phase's stated success gate ("kill -9 mid-recording -> playable file missing <= ~10 s, verified empirically").

Purpose: `movieFragmentInterval` crash safety cannot be verified by XCTest (an XCTest process cannot cleanly `kill -9` itself and still assert). RESEARCH.md prescribes a shell-driven gate against a minimal recording harness. This proves the fragmenting mechanism on the local OS; it is flagged for a mandatory re-run on the 14.2 floor before release.

Output: `ScreenRecorderHarness.swift` (record + validate modes via launch args), a DEBUG launch-argument dispatch in the app entry, and `Scripts/kill9-recovery-gate.sh`.
</objective>
