<objective>
Clean up stale aggregate device UIDs from previous Caddie sessions on app launch.

Purpose: If Caddie crashes or is force-quit while recording, the aggregate device created for system audio capture is never destroyed. These orphaned devices accumulate in CoreAudio's device list. Cleaning them up on startup prevents resource leaks and potential conflicts with new recordings.

Output: Static cleanup method on SystemAudioCapture, called during AppState.initialize() before coordinator starts.
</objective>

    Add a static method `cleanupStaleAggregateDevices()` to SystemAudioCapture that:

    1. Uses `kAudioHardwarePropertyDevices` on `kAudioObjectSystemObject` to get all AudioDeviceIDs
    2. For each device, reads `kAudioDevicePropertyDeviceUID` (CFString)
    3. Checks if the UID string starts with "com.caddie.systemTap."
    4. For each matching device, calls `AudioHardwareDestroyAggregateDevice(deviceID)`
    5. Logs via Logger: "Cleaned up N stale aggregate device(s)" if N > 0
    6. Logs nothing if N == 0 (silent success)
    7. Does NOT throw -- cleanup is best-effort. Log errors but continue.

    The UID prefix "com.caddie.systemTap." matches the pattern on line 160 of SystemAudioCapture.swift.

    Implementation pattern for CoreAudio property reads:
    ```swift
    var address = AudioObjectPropertyAddress(

    var dataSize: UInt32 = 0
    AudioObjectGetPropertyDataSize(AudioObjectID(kAudioObjectSystemObject), &address, 0, nil, &dataSize)
    let deviceCount = Int(dataSize) / MemoryLayout<AudioDeviceID>.size
    var deviceIDs = [AudioDeviceID](repeating: 0, count: deviceCount)
    AudioObjectGetPropertyData(AudioObjectID(kAudioObjectSystemObject), &address, 0, nil, &dataSize, &deviceIDs)
    ```
