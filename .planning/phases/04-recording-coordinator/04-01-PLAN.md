<feature>
  <name>RecordingState pure state machine</name>
  <files>Sources/Coordinator/RecordingState.swift, Tests/RecordingStateTests.swift</files>
  <behavior>
    Valid transitions (reduce returns non-nil):
    - idle + meetingDetected -> recording (returns .startRecording side effect)
    - recording + meetingEnded -> transcribing (returns .stopAndTranscribe side effect)
    - recording + recordingFailed -> error (returns .notifyError side effect)
    - transcribing + transcriptionComplete -> idle (returns .notifyComplete side effect)
    - transcribing + transcriptionFailed -> error (returns .notifyError side effect)
    - error + retryRequested -> transcribing (returns .retryTranscription side effect)
    - error + reset -> idle (returns nil, just resets state)

    Invalid transitions (reduce returns nil, state unchanged):
    - idle + meetingEnded -> nil (not recording)
    - idle + transcriptionComplete -> nil (not transcribing)
    - idle + retryRequested -> nil (no error to retry)
    - recording + meetingDetected -> nil (already recording)
    - recording + transcriptionComplete -> nil (not transcribing yet)
    - transcribing + meetingDetected -> nil (busy transcribing)
    - transcribing + meetingEnded -> nil (already stopped)

    State associated values:
    - recording carries meetingId: String
    - transcribing carries meetingId: String
    - error carries meetingId: String, Error
  </behavior>
  <implementation>
    RecordingState enum: Sendable, Equatable (for tests). Four cases with associated values.

    RecordingEvent enum: Sendable. Cases: meetingDetected(DetectedMeeting), meetingEnded, recordingFailed(Error), transcriptionComplete(meetingId: String), transcriptionFailed(meetingId: String, Error), retryRequested(meetingId: String), reset.

    RecordingSideEffect enum: Sendable. Cases: startRecording(meetingId: String, meeting: DetectedMeeting), stopAndTranscribe(meetingId: String), retryTranscription(meetingId: String), notifyComplete(meetingId: String), notifyError(meetingId: String, Error).

    Free function or static method: `RecordingState.reduce(state:event:) -> (newState: RecordingState, sideEffect: RecordingSideEffect?)?` -- returns nil tuple for invalid transitions, or (newState, optionalSideEffect) for valid ones.

    The reduce function is a pure switch over (state, event) pairs. No async, no side effects, no imports beyond Foundation. This is the contract that RecordingCoordinator will call synchronously.

    Note: DetectedMeeting is defined in MeetingDetector.swift. It has app: String, title: String, processId: pid_t?. The state machine needs to reference it but does not modify it.

    For meetingId generation: the reduce function receives a meetingId parameter in the meetingDetected event or generates one. Keep it simple -- let the caller generate the meetingId and pass it via the event.

    Update project.yml: add Sources/Coordinator/ to the Caddie target's sources list. Follow the existing pattern in project.yml for source groups.
  </implementation>
</feature>
