<objective>
Close the pipeline-to-coordinator feedback gap: TranscriptionPipeline.enqueue() fires-and-forgets,
so the coordinator state machine is stuck in .transcribing forever after a meeting ends.

Purpose: Complete the REC-03 lifecycle (idle -> recording -> transcribing -> done/error) by wiring
pipeline completion back to the coordinator's state machine.

Output: Pipeline calls back to coordinator on success/failure, coordinator exits .transcribing state.
</objective>

  <action>
    1. In `executeStopAndTranscribe(meetingId:)`, replace the fire-and-forget enqueue call
       (line 180) with one that passes a completion callback:
       ```swift
       await pipeline.enqueue(meetingId: meetingId, database: db) { [weak self] meetingId, result in
           guard let self else { return }
           Task {
               switch result {
               case .success:
                   await self.handle(.transcriptionComplete(meetingId: meetingId))
               case .failure(let error):
                   await self.handle(.transcriptionFailed(meetingId: meetingId, error))
               }
           }
       }
       ```
       NOTE: `[weak self]` is not valid on actors -- actors do not support weak references.
       Instead, capture `self` directly (actor references are already safe -- they are reference
       types with isolated state). The closure is `@Sendable` and calls back into the actor
       via `await self.handle()`, which is correct cross-actor communication:
       ```swift
       await pipeline.enqueue(meetingId: meetingId, database: db) { [self] meetingId, result in
           Task {
               switch result {
               case .success:
                   await self.handle(.transcriptionComplete(meetingId: meetingId))
               case .failure(let error):
                   await self.handle(.transcriptionFailed(meetingId: meetingId, error))
               }
           }
       }
       ```
