  <action>
    Create RecordingCoordinator as a Swift actor with these properties:
    - private(set) var state: RecordingState = .idle
    - private let recorder: AudioRecorder
    - private let pipeline: TranscriptionPipeline
    - private let database: AppDatabase
    - private let detector: MeetingDetector
    - var onStateChange: (@Sendable (RecordingState) -> Void)?
    - private let logger: Logger

    Constructor takes: database (non-optional -- fail fast per ARCHITECTURE.md anti-pattern 3), recorder, pipeline, detector. All required. This eliminates the init race (REC-04) -- coordinator cannot be created until all dependencies exist.

    Public method: func handle(_ event: RecordingEvent) async
      1. Call RecordingState.reduce(state: state, event: event)
      2. If nil (invalid transition), log warning, return
      3. Update self.state = result.newState
      4. Call onStateChange?(state) for UI updates
      5. If result.sideEffect != nil, call execute(sideEffect)

    Private method: func execute(_ effect: RecordingSideEffect) async
      Switch on effect:
      - .startRecording(meetingId, meeting):
        Generate meeting record, insert into DB (do-catch, NOT try?), start recorder
        If DB insert fails, transition to error state
      - .stopAndTranscribe(meetingId):
        Stop recorder, update DB status to transcribing, enqueue pipeline
        Pipeline completion callback: handle(.transcriptionComplete) or handle(.transcriptionFailed)
      - .retryTranscription(meetingId):
        Validate WAV exists, update DB status, re-enqueue pipeline
      - .notifyComplete(meetingId):
        Update DB status to done (log only)
      - .notifyError(meetingId, error):
        Update DB status to error with error message

    Public method: func start() -- wires detector callbacks and starts detector:
      detector.onMeetingStarted = { meeting in Task { await self.handle(.meetingDetected(meeting)) } }
      detector.onMeetingEnded = { Task { await self.handle(.meetingEnded) } }
      detector.start()

    Public method: func stop() -- stops detector, stops recorder if recording

    Public convenience methods that forward to handle():
      func retryTranscription(meetingId: String) async
      func stopRecording() async

    For tests: Create RecordingCoordinatorTests using in-memory AppDatabase, MockASREngine, MockDiarizationEngine (from existing test mocks). Verify state transitions via the coordinator's state property. Use polling pattern (100ms intervals, 10s timeout) established in Phase 02.
